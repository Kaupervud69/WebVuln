* Классифицкация уязвимостей раскрытия информации
    * Неудаление внутреннего контента из общедоступного контента. Например, комментарии разработчиков в разметке иногда видны пользователям в рабочей среде.
    * Небезопасная конфигурация веб-сайта и связанных с ним технологий. Невыключение функций отладки и диагностики, конфигурации по умолчанию.
    * Несовершенный дизайн и поведение приложения. Ввеб-сайт возвращает разные ответы при возникновении разных состояний ошибки.

# Методология

## Фаззинг

* Отправить неожиданные данные
* Cравниnm коды состояния HTTP, время ответа, длину и т. д.

* [расширение Logger++](https://github.com/portswigger/logger-plus-plus) - позволяет определять расширенные фильтры для выделения интересных записей.

## Обычные источники раскрытия информации

* **Файлы для веб-сканеров**
  * /robots.txt и /sitemap.xml
 
* **Списки каталогов**
> если веб-сайт не реализует надлежащий контроль доступа, утечка существования и местоположения конфиденциальных ресурсов.
 
* **Комментарии разработчиков**

> они могут намекать на существование скрытых каталогов или давать подсказки о логике приложения.

* **Сообщения об ошибках**

* может раскрывать информацию о том, какой ввод или тип данных ожидается от данного параметра.
* могут предоставлять информацию о различных технологиях, используемых веб-сайтом.
* изучить общедоступный исходный код если веб-сайт использует какой-то фреймворк с открытым исходным кодом.

* **Отладочные данные**

* могут содержать
  * Значения ключевых переменных сеанса, которые можно изменять с помощью ввода пользователя
  * Имена хостов и учетные данные для внутренних компонентов
  * Имена файлов и каталогов на сервере
  * Ключи, используемые для шифрования данных, передаваемых через клиент

* **Страницы учетной записи пользователя**

> Пользователь может не иметь возможности загрузить страницу учетной записи другого пользователя полностью, но логика извлечения и отображения зарегистрированного адреса электронной почты пользователя, например, может не проверять, соответствует ли параметр пользователя пользователю, который в данный момент вошел в систему. В этом случае простое изменение параметра, позволит отображать адреса электронной почты произвольных пользователей на странице своей учетной записи.
