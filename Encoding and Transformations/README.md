**Кодирование и преобразования**

Кодирование и преобразования — это техники, которые изменяют способ представления или передачи данных без изменения их основного смысла. Распространенные примеры включают URL-кодирование, Base64, кодирование HTML-сущностей и преобразования Unicode. Атакующие используют эти методы как инструменты для обхода фильтров ввода, уклонения от межсетевых экранов веб-приложений (WAF) или выхода из процедур очистки данных.

**Unicode**

Unicode — это универсальный стандарт кодирования символов, используемый для представления текста практически из всех систем письменности в мире. Каждому символу (буквы, цифры, символы, эмодзи) присваивается уникальная кодовая точка (например, U+0041 для "A"). Форматы кодирования Unicode, такие как UTF-8 и UTF-16, определяют, как эти кодовые точки хранятся в виде байтов.

**Нормализация Unicode**

Нормализация Unicode — это процесс преобразования текста Unicode в стандартизированную, согласованную форму, чтобы эквивалентные символы представлялись в памяти одинаково.

**Справочная таблица форм нормализации Unicode**

*   **NFC (Форма нормализации C — каноническая композиция):** Объединяет разложенные последовательности в предварительно составленные символы, где это возможно.
*   **NFD (Форма нормализации D — каноническое разложение):** Разбивает символы на их разложенные формы (основа + комбинируемые знаки).
*   **NFKC (Форма нормализации KC — совместимая композиция):** Похожа на NFC, но также заменяет символы на их эквиваленты совместимости (может изменить внешний вид или формат).
*   **NFKD (Форма нормализации KD — совместимое разложение):** Похожа на NFD, но также разлагает символы совместимости.

| Символ | Полезная нагрузка | После нормализации |
| :--- | :--- | :--- |
| ‥ (U+2025) | ‥/‥/‥/etc/passwd | ../../../etc/passwd |
| ︰ (U+FE30) | ︰/︰/︰/etc/passwd | ../../../etc/passwd |
| ＇ (U+FF07) | ＇ or ＇1＇=＇1 | ' or '1'='1 |
| ＂ (U+FF02) | ＂ or ＂1＂=＂1 | " or "1"="1 |
| ﹣ (U+FE63) | admin'﹣﹣ | admin'-- |
| 。 (U+3002) | domain。com | domain.com |
| ／ (U+FF0F) | ／／domain.com | //domain.com |
| ＜ (U+FF1C) | ＜img src=a＞ | <img src=a/> |
| ﹛ (U+FE5B) | ﹛﹛3+3﹜﹜ | {{3+3}} |
| ［ (U+FF3B) | ［［5+5］］ | [[5+5]] |
| ＆ (U+FF06) | ＆＆whoami | &&whoami |
| ｐ (U+FF50) | shell.ｐʰｐ | shell.php |
| ʰ (U+02B0) | shell.ｐʰｐ | shell.php |
| ª (U+00AA) | ªdmin | admin |

```python
import unicodedata
string = "ᴾᵃʸˡᵒᵃᵈˢ𝓐𝓵𝓵𝕋𝕙𝕖𝒯𝒽𝒾𝓃ℊ𝓈"
print ('NFC: ' + unicodedata.normalize('NFC', string))
print ('NFD: ' + unicodedata.normalize('NFD', string))
print ('NFKC: ' + unicodedata.normalize('NFKC', string))
print ('NFKD: ' + unicodedata.normalize('NFKD', string))
```

**Punycode**

Punycode — это способ представления символов Unicode (включая не-ASCII буквы, символы и письменности) с использованием только ограниченного набора ASCII-символов (буквы, цифры и дефисы).

В основном он используется в системе доменных имен (DNS), которая традиционно поддерживает только ASCII. Punycode позволяет использовать интернационализированные доменные имена (IDN), чтобы доменные имена могли включать символы из многих языков путем их преобразования в безопасную ASCII-форму.

| Видимо в браузере (поддержка IDN) | Фактический ASCII (Punycode) |
| :--- | :--- |
| раypal.com | xn--ypal-43d9g.com |
| paypal.com | paypal.com |

В MySQL похожие символы считаются равными. Такое поведение может быть использовано во вред в разделах сброса пароля, восстановления пароля и у провайдеров OAuth.

```sql
SELECT 'a' = 'ᵃ';
+-------------+
| 'a' = 'ᵃ'   |
+-------------+
|           1 |
+-------------+
```

Этот трюк срабатывает, если SQL-запрос использует `COLLATE utf8mb4_0900_as_cs`.

```sql
SELECT 'a' = 'ᵃ' COLLATE utf8mb4_0900_as_cs;
+----------------------------------------+
| 'a' = 'ᵃ' COLLATE utf8mb4_0900_as_cs   |
+----------------------------------------+
|                                      0 |
+----------------------------------------+
```

**Base64**

Кодировка Base64 — это метод преобразования двоичных данных (например, изображений или файлов) или текста со специальными символами в читаемую строку, использующую только ASCII-символы (A-Z, a-z, 0-9, + и /). Каждые 3 байта входных данных делятся на 4 группы по 6 бит и сопоставляются с 4 символами Base64. Если входные данные не кратны 3 байтам, вывод дополняется символами `=`.

```bash
echo -n admin | base64                            
YWRtaW4=

echo -n YWRtaW4= | base64 -d
admin
```
