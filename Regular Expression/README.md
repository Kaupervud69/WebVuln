**Регулярные выражения**

ReDoS (Regular Expression Denial of Service — отказ в обслуживании через регулярные выражения) — это тип атаки, который использует тот факт, что некоторые регулярные выражения могут обрабатываться чрезвычайно долго, вызывая неполадки в работе приложений или сервисов вплоть до их полной остановки или краха.

**Содержание**

*   Инструменты
*   Методология
    *   "Злые" регулярные выражения (Evil Regex)
    *   Ограничение на количество шагов возврата (Backtrack Limit)
*   Ссылки

**Инструменты**

*   **tjenkinson/redos-detector** — интерфейс командной строки (CLI) и библиотека, которая с высокой точностью проверяет, безопасно ли регулярное выражение от атак ReDoS. Поддерживается в браузере, Node.js и Deno.
*   **doyensec/regexploit** — инструмент для поиска регулярных выражений, уязвимых для ReDoS (отказ в обслуживании через регулярные выражения).
*   **devina.io/redos-checker** — онлайн-инструмент для проверки регулярных выражений на наличие уязвимостей к DoS-атакам.

**Методология**

**"Злые" регулярные выражения (Evil Regex)**

"Злое" регулярное выражение содержит:

*   Группировку с повторением.
*   Внутри повторяющейся группы:
    *   Повторение.
    *   Альтернативы, которые перекрываются.

**Примеры таких выражений:**

*   `(a+)+`
*   `([a-zA-Z]+)*`
*   `(a|aa)+`
*   `(a|a?)+`
*   `(.*a){x}` для `x > 10`

Эти регулярные выражения могут быть атакованы с помощью строки, состоящей из множества повторяющихся символов, за которыми следует символ, не подходящий под шаблон. Например: `aaaaaaaaaaaaaaaaaaaa!` (20 символов 'a', за которыми следует '!').

Для такого ввода движок регулярных выражений попытается перебрать все возможные способы группировки символов 'a', прежде чем поймет, что совпадение в конечном итоге невозможно из-за символа `!`. Это приводит к взрывному росту количества попыток возврата (backtracking).

**Ограничение на количество шагов возврата (Backtrack Limit)**

Возврат (backtracking) в регулярных выражениях происходит, когда движок пытается сопоставить шаблон и встречает несоответствие. Движок затем возвращается к предыдущей успешной позиции и пробует альтернативный путь для поиска совпадения. Этот процесс может повторяться множество раз, особенно со сложными шаблонами и большими входными строками.

**Параметры конфигурации PCRE в PHP:**

| Название | Значение по умолчанию | Примечание |
| :--- | :--- | :--- |
| `pcre.backtrack_limit` | 1 000 000 | 100 000 для PHP < 5.3.7 |
| `pcre.recursion_limit` | 100 000 | |
| `pcre.jit` | 1 | |

Иногда можно заставить регулярное выражение превысить лимит в 100 000 рекурсий, что вызовет ReDoS и заставит функцию `preg_match` вернуть `false`:

```php
$pattern = '/(a+)+$/';
$subject = str_repeat('a', 1000) . 'b';

if (preg_match($pattern, $subject)) {
    echo "Совпадение найдено";
} else {
    echo "Совпадений нет";
}
```
