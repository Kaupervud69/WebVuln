**Внешняя модификация переменных**

Уязвимость внешней модификации переменных возникает, когда веб-приложение некорректно обрабатывает пользовательский ввод, позволяя злоумышленникам перезаписывать внутренние переменные. В PHP такие функции, как `extract($_GET)`, `extract($_POST)` или `import_request_variables()`, могут быть использованы во вред, если они импортируют контролируемые пользователем данные в глобальную область видимости без надлежащей проверки. Это может привести к таким проблемам безопасности, как несанкционированное изменение логики приложения, повышение привилегий или обход механизмов безопасности.

**Краткое содержание**

*   **Методология**
    *   Перезапись критически важных переменных
    *   Отравление подключения файлов (File Inclusion)
    *   Внедрение глобальных переменных
*   **Меры по устранению**
*   **Ссылки**

**Методология**

Функция `extract()` в PHP импортирует переменные из массива в текущую таблицу символов. Хотя это может показаться удобным, это может привести к серьезным рискам безопасности, особенно при обработке данных, предоставленных пользователем.

*   Она позволяет перезаписывать существующие переменные.
*   Это может привести к "загрязнению" переменных, влияя на механизмы безопасности.
*   Это может быть использовано как "гаджет" для запуска других уязвимостей, таких как удаленное выполнение кода (RCE) и локальное включение файлов (LFI).

По умолчанию `extract()` использует флаг `EXTR_OVERWRITE`, что означает замену существующих переменных, если их имена совпадают с ключами во входном массиве.

**Перезапись критически важных переменных**

Если `extract()` используется в скрипте, который полагается на определенные переменные, злоумышленник может ими манипулировать.

```php
<?php
    $authenticated = false;
    extract($_GET);
    if ($authenticated) {
        echo "Доступ разрешен!";
    } else {
        echo "Доступ запрещен!";
    }
?>
```

**Эксплуатация:**

В этом примере использование `extract($_GET)` позволяет злоумышленнику установить переменную `$authenticated` в значение `true`:

`http://example.com/vuln.php?authenticated=true`
`http://example.com/vuln.php?authenticated=1`

**Отравление подключения файлов (Poisoning File Inclusion)**

Если `extract()` используется вместе с подключением файлов, атакующие могут контролировать пути к файлам.

```php
<?php
    $page = "config.php";
    extract($_GET);
    include "$page";
?>
```

**Эксплуатация:**

`http://example.com/vuln.php?page=../../etc/passwd`

**Внедрение глобальных переменных (Global Variable Injection)**

⚠️ Начиная с PHP 8.1.0, запись во весь массив `$GLOBALS` больше не поддерживается.

Перезапись `$GLOBALS`, когда приложение вызывает функцию `extract` для недоверенного значения:

```php
extract($_GET);
```

Злоумышленник может манипулировать глобальными переменными:

`http://example.com/vuln.php?GLOBALS[admin]=1`

**Меры по устранению**

Используйте флаг `EXTR_SKIP`, чтобы предотвратить перезапись:

```php
extract($_GET, EXTR_SKIP);
```
