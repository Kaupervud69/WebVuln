**Загрязнение HTTP-параметров (HTTP Parameter Pollution)**

HTTP Parameter Pollution (HPP) — это техника обхода веб-атак, которая позволяет злоумышленнику сформировать HTTP-запрос таким образом, чтобы манипулировать логикой веб-приложения или извлечь скрытую информацию. Эта техника обхода основана на разделении вектора атаки между несколькими экземплярами одного и того же параметра (`?param1=value&param1=value`). Поскольку не существует формального способа разбора HTTP-параметров, отдельные веб-технологии имеют свои уникальные способы разбора и чтения параметров URL с одинаковыми именами. Некоторые берут первое вхождение, некоторые — последнее, а некоторые интерпретируют их как массив. Это поведение используется злоумышленником для обхода механизмов безопасности, основанных на поиске закономерностей (сигнатур).

**Методология**

HTTP Parameter Pollution (HPP) — это уязвимость веб-безопасности, при которой злоумышленник внедряет несколько экземпляров одного и того же HTTP-параметра в запрос. Поведение сервера при обработке дублирующихся параметров может различаться, что потенциально может привести к неожиданному или эксплуатируемому поведению.

HPP может быть направлена на два уровня:

*   **Клиентский HPP (Client-Side HPP)**: Эксплуатирует JavaScript-код, выполняющийся на стороне клиента (браузере).
*   **Серверный HPP (Server-Side HPP)**: Эксплуатирует то, как сервер обрабатывает несколько параметров с одинаковым именем.

**Примеры:**

```
/app?debug=false&debug=true
/transfer?amount=1&amount=5000
```

**Таблица поведения при загрязнении параметров**

Когда в запросе передано `?par1=a&par1=b`:

| Технология | Результат разбора | Итог (par1=) |
| :--- | :--- | :--- |
| ASP.NET/IIS | Все вхождения | `a,b` |
| ASP/IIS | Все вхождения | `a,b` |
| Golang net/http - `r.URL.Query().Get("param")` | Первое вхождение | `a` |
| Golang net/http - `r.URL.Query()["param"]` | Все вхождения в виде массива | `['a','b']` |
| IBM HTTP Server | Первое вхождение | `a` |
| IBM Lotus Domino | Первое вхождение | `a` |
| JSP, Servlet/Tomcat | Первое вхождение | `a` |
| mod_wsgi (Python)/Apache | Первое вхождение | `a` |
| Nodejs | Все вхождения | `a,b` |
| Perl CGI/Apache | Первое вхождение | `a` |
| PHP/Apache | Последнее вхождение | `b` |
| PHP/Zues | Последнее вхождение | `b` |
| Python Django | Последнее вхождение | `b` |
| Python Flask | Первое вхождение | `a` |
| Python/Zope | Все вхождения в виде массива | `['a','b']` |
| Ruby on Rails | Последнее вхождение | `b` |

**Полезные нагрузки для загрязнения параметров**

**Дублирующиеся параметры:**

```
param=value1&param=value2
```

**Внедрение массива (Array Injection):**

```
param[]=value1
param[]=value1&param[]=value2
param[]=value1&param=value2
param=value1&param[]=value2
```

**Закодированная инъекция (Encoded Injection):**

```
param=value1%26other=value2
```

**Вложенная инъекция (Nested Injection):**

```
param[key1]=value1&param[key2]=value2
```

**JSON-инъекция (JSON Injection):**

```json
{
    "test": "user",
    "test": "admin"
}
```
