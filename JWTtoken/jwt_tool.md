# Использование jwt_tool.py

```bash
jwt_tool.py [-h] [-b] [-t TARGETURL] [-r REQUEST] [-rt RATE] [-i]
            [-rc COOKIES] [-rh HEADERS] [-pd POSTDATA] [-cv CANARYVALUE]
            [-np] [-nr] [-M MODE] [-X EXPLOIT] [-ju JWKSURL] [-S SIGN]
            [-pr PRIVKEY] [-T] [-I] [-hc HEADERCLAIM] [-pc PAYLOADCLAIM]
            [-hv HEADERVALUE] [-pv PAYLOADVALUE] [-C] [-d DICT]
            [-p PASSWORD] [-kf KEYFILE] [-V] [-pk PUBKEY] [-jw JWKSFILE]
            [-Q QUERY] [-v]
            [jwt]
```

## Позиционные аргументы

| Аргумент | Описание |
|----------|----------|
| `jwt` | JWT для обработки (не требуется указывать, если токен находится в заголовках или cookies) |

## Опции

### Общие опции
| Опция | Описание |
|-------|----------|
| `-h, --help` | Показать это справочное сообщение |
| `-b, --bare` | Возвращать ТОЛЬКО ТОКЕНЫ |
| `-v, --verbose` | При разборе и выводе создавать (немного) более подробный вывод |

### Опции HTTP-запросов
| Опция | Описание |
|-------|----------|
| `-t, --targeturl TARGETURL` | URL для отправки HTTP-запроса с новым JWT |
| `-r, --request REQUEST` | Базовый URL запроса |
| `-rt, --rate RATE` | Макс. количество запросов в минуту |
| `-i, --insecure` | Использовать HTTP для переданного запроса |
| `-rc, --cookies COOKIES` | Cookies для отправки с поддельным HTTP-запросом |
| `-rh, --headers HEADERS` | Заголовки для отправки с поддельным HTTP-запросом (можно использовать несколько раз для дополнительных заголовков) |
| `-pd, --postdata POSTDATA` | Текстовая строка с данными для POST-запроса |
| `-cv, --canaryvalue CANARYVALUE` | Текстовая строка, которая появляется в ответе для валидного токена (например, "Welcome, ticarpi") |
| `-np, --noproxy` | Отключить прокси для текущего запроса (изменить в jwtconf.ini для постоянного отключения) |
| `-nr, --noredir` | Отключить перенаправления для текущего запроса (изменить в jwtconf.ini для постоянного отключения) |

### Режимы сканирования
| Опция | Описание |
|-------|----------|
| `-M, --mode MODE` | Режим сканирования:<br>• `pb` – аудит по плейбуку<br>• `er` – фаззинг существующих утверждений для вызова ошибок<br>• `cc` – фаззинг распространённых утверждений<br>• `at` – Все тесты! |

### Эксплуатация уязвимостей
| Опция | Описание |
|-------|----------|
| `-X, --exploit EXPLOIT` | Эксплуатация известных уязвимостей:<br>• `a` – alg:none<br>• `n` – нулевая подпись<br>• `b` – пустой пароль принимается в подписи<br>• `p` – 'психическая подпись' принимается при ECDSA-подписи<br>• `s` – подделка JWKS (укажите JWKS URL с `-ju` или задайте в jwtconf.ini для автоматизации атаки)<br>• `k` – путаница ключей (укажите публичный ключ с `-pk`)<br>• `i` – внедрение встроенного JWKS |
| `-ju, --jwksurl JWKSURL` | URL-адрес для размещения поддельного JWKS |

### Подпись токенов
| Опция | Описание |
|-------|----------|
| `-S, --sign SIGN` | Подписать результирующий токен:<br>• `hs256/hs384/hs512` – HMAC-SHA подпись (укажите секрет с `-k/-p`)<br>• `rs256/rs384/rs512` – RSA подпись (укажите RSA приватный ключ с `-pr`)<br>• `es256/es384/es512` – подпись на эллиптических кривых (укажите EC приватный ключ с `-pr`)<br>• `ps256/ps384/ps512` – PSS-RSA подпись (укажите RSA приватный ключ с `-pr`) |
| `-pr, --privkey PRIVKEY` | Приватный ключ для асимметричной криптографии |

### Изменение содержимого JWT
| Опция | Описание |
|-------|----------|
| `-T, --tamper` | Изменить содержимое JWT (задайте параметры подписи с `-S` или используйте эксплойты с `-X`) |
| `-I, --injectclaims` | Внедрить новые утверждения и обновить существующие новыми значениями (задайте параметры подписи с `-S` или используйте эксплойты с `-X`) (задайте целевое утверждение с `-hc/-pc` и значения/списки для внедрения с `-hv/-pv`) |
| `-hc, --headerclaim HEADERCLAIM` | Утверждение заголовка для изменения |
| `-pc, --payloadclaim PAYLOADCLAIM` | Утверждение полезной нагрузки для изменения |
| `-hv, --headervalue HEADERVALUE` | Значение (или файл со значениями) для внедрения в изменённое утверждение заголовка |
| `-pv, --payloadvalue PAYLOADVALUE` | Значение (или файл со значениями) для внедрения в изменённое утверждение полезной нагрузки |

### Взлом и проверка
| Опция | Описание |
|-------|----------|
| `-C, --crack` | Взломать ключ для HMAC-SHA токена (укажите `-d/-p/-kf`) |
| `-d, --dict DICT` | Файл словаря для взлома |
| `-p, --password PASSWORD` | Пароль для взлома |
| `-kf, --keyfile KEYFILE` | Файл ключей для взлома (при атаках с подписанным 'kid') |
| `-V, --verify` | Проверить подпись RSA с помощью публичного ключа (укажите `-pk/-jw`) |
| `-pk, --pubkey PUBKEY` | Публичный ключ для асимметричной криптографии |
| `-jw, --jwksfile JWKSFILE` | Хранилище JSON Web Key для асимметричной криптографии |

### Запросы и логи
| Опция | Описание |
|-------|----------|
| `-Q, --query QUERY` | Запросить ID токена в файле журнала для просмотра деталей запроса (например, `-Q jwttool_46820e62fe25c10a3f5498e426a9f03a`) |
